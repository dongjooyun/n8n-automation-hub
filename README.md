## 내 로직의 주인은 나여야 하니까

자동화의 미래는 단순히 효율을 높이는 것에 그치지 않습니다. 누가 그 로직을 소유하고 통제하느냐의 문제입니다. 남의 컴퓨터(Public Cloud)를 빌려 쓰는 시대에서 벗어나, 우리는 직접 인프라의 주권을 되찾고 보안과 자유도를 우리 방식대로 정의합니다. 이 서버는 단순히 워크플로우를 돌리는 도구가 아니라, 우리 팀의 로직을 안전하게 보관하는 요새입니다.

### 1. 설치 환경: 홈메이드 인프라의 맛

클라우드 구독 버튼을 누르는 쉬운 길 대신, 직접 땅을 다지고 기둥을 세우는 길을 택했습니다.

- **OS**: Ubuntu 24.04 LTS (WSL2 기반의 독립적인 환경입니다.)
- **Runtime**: Docker & Docker Compose (복제가 가능하고 깔끔합니다.)
- **Security**: Cloudflare Tunnel (공인 IP를 숨기는 안전한 비밀 통로입니다.)
- **Access Port**: 1616 (보편적인 5678 포트 대신 고유 포트를 할당했습니다.)

**왜 굳이 On-premises인가요?**
데이터 주권은 구호가 아닙니다. API 키와 민감한 워크플로우가 제3자의 서버가 아닌 오직 우리 서버 내부에만 존재할 때 진정한 보안이 시작됩니다. 또한 무거운 AI 워크플로우를 구동할 때 월말 영수증을 걱정하지 않아도 된다는 점은 무엇과도 바꿀 수 없는 매력입니다.

### 2. 폴더 구조 분석: 이 시스템의 설계도

코드로 관리되는 인프라(IaC)는 정돈된 디렉토리에서 시작됩니다.

- **docker-compose.yml**: 서비스의 전체 생명주기를 정의한 설계도입니다.
- **n8n_data/**: 컨테이너가 교체되어도 우리의 데이터는 남아야 합니다. 모든 휘발성 데이터를 로컬에 보존하는 영구 저장소입니다.
- **README.md**: 지금 읽고 계신 운영 지침서입니다. 미래의 내가 길을 잃지 않게 도와주는 친절한 가이드입니다.

### 3. 자동화 활용 계획: 우리가 꿈꾸는 효율성

반복 업무는 n8n에게 맡기고, 사람은 더 가치 있는 고민에 집중합니다.

1. **마케팅 OSMU 전략 기반 콘텐츠 자동 변환**
   - 유튜브 롱폼 영상을 소스로 삼아 인사이트 리포트, 이메일, 인스타그램, 커뮤니티 홍보글, 랜딩페이지 문구까지 자동으로 생성합니다. 원소스 멀티유즈의 정점을 찍는 워크플로우입니다.
2. **Notion-Slack 연동 프로젝트 티켓 관리**
   - 노션의 상태 변화를 실시간으로 감지하여 슬랙으로 공유합니다. 사람이 일일이 체크하지 않아도 팀의 싱크가 맞춰지는 경험을 제공합니다.
3. **기능 명세 기반 TC 작성 및 동기화**
   - 피그마와 노션의 명세를 분석하여 테스트 케이스를 자동 작성합니다. 기획이 바뀌면 테스트 문서도 알아서 따라오는 자동화를 지향합니다.

### 4. 정량적 기대 효과: 신입 PM의 시간은 소중하니까

국내 주니어 기획자(0년 차) 평균 연봉 기준 시급(20,000원)으로 환산한 결과입니다.

| 업무 항목                | 수동 작업 시간 | 자동화 후 시간 | 주간 절감 시간  |
| :----------------------- | :------------- | :------------- | :-------------- |
| OSMU 콘텐츠 변형/배포    | 10시간         | 1시간          | 9시간           |
| 프로젝트 티켓 관리/공유  | 3시간          | 0.5시간        | 2.5시간         |
| TC 작성 및 명세 업데이트 | 4시간          | 0.5시간        | 3.5시간         |
| **합계**                 | **17시간**     | **2시간**      | **15시간 절감** |

- **시간적 가치**: 주당 15시간 절감 (월 60시간 / 연 720시간)
- **경제적 가치**: 시급 20,000원 기준, **연간 약 1,440만 원**의 비용 절감 효과

### 5. 소스 출처: 최신성을 유지하는 이유

- **Base Image**: `n8nio/n8n:latest`
- **선택 이유**: AI Agent 구현은 빠르게 변화하는 생태계입니다. 최신 AI 노드와 보안 패치를 즉각 반영하기 위해 공식 최신 릴리스를 선택했습니다.

### 6. 작지만 소중한 트러블슈팅: 오타와 권한의 사투

- **파일명 오류**: `docker-composer.yml`로 오기입하여 Docker가 파일을 인식하지 못하는 문제를 발견하고 수정했습니다.
- **권한 이슈**: 컨테이너가 볼륨에 접근하지 못하는 현상을 `chown -R 1000:1000` 명령어로 해결했습니다.

## n8n On-Premises 자동화 서버 구축

### 1. 설치 환경

안정적인 워크플로우 실행과 데이터 보안을 위해 클라우드 의존성 제거한 독립 환경 구축

- **OS**: Ubuntu 24.04 LTS (WSL2)
- **Runtime**: Docker / Docker Compose
- **Security**: Cloudflare Tunnel (HTTPS Proxy)
- **Access Port**: 1616 (내부 충돌 방지, 보안 강화 목적)

**구축 목적:**

- 데이터 소유권 확보를 통해 외부 보안 리스크 차단
- AI 워크플로우 실행에 따른 클라우드 인프라 비용 부담을 제거하여 지속 가능한 자동화 환경 조성

### 2. 디렉토리 구조

기술적 재현성과 데이터 보존을 위한 구조

- **docker-compose.yml**: 인프라 구성을 코드로 관리(IaC)하여 서비스 배포 및 확장 일관성 보장
- **n8n_data/**: Persistent Volume 설정을 통해 컨테이너 생명주기와 관계없이 워크플로우 및 사용자 데이터를 영구 보존
- **README.md**: 시스템 아키텍처 및 자동화 로드맵 기록

### 3. 자동화 활용 계획

기획 및 마케팅 업무의 반복 공수를 최소화하기 위한 자동화 계획입니다.

1. **마케팅 OSMU 전략 자동화**: 생성된 핵심 콘텐츠를 유튜브 롱폼 소스, 인사이트, 이메일, 인스타그램, 커뮤니티 홍보글 등 채널별 특성에 맞는 콘텐츠로 자동 변형 및 배포
2. **협업 도구 데이터 동기화**: Notion의 프로젝트 상태 변화를 Slack으로 실시간 전파하여 커뮤니케이션 지연 방지
3. **품질 관리(QA) 자동화**: 기획서 및 Figma 명세를 추출하여 테스트 케이스(TC)를 자동 생성하고 상호 연결 상태 실시간 업데이트(Playwright 활용하여 E2E 자동화까지가 목표)

### 4. 정량적 기대 효과

국내 주니어 기획자 평균 초봉 기준 시급(20,000원) 적용

| 평가지표            | 자동화 전 추정(주) | 자동화 후(주) | 절감 시간(주) |
| :------------------ | :----------------- | :------------ | :------------ |
| 콘텐츠 다각화 작업  | 10시간             | 1시간         | 9시간         |
| 프로젝트 현황 관리  | 3시간              | 0.5시간       | 2.5시간       |
| 명세 업데이트 및 QA | 4시간              | 0.5시간       | 3.5시간       |
| **합계**            | **17시간**         | **2시간**     | **15시간**    |

- **시간 절감**: 주당 15시간 (연간 약 720시간) 절감
- **경제 효과**: 시급 20,000원 산정 시 **연간 약 1,440만 원**의 이득 발생

### 5. 소스 출처 및 버전 정보

- **Source**: `n8nio/n8n:latest` (Official Docker Image)
- **선택 이유**: AI Agent 노드 활용 및 최신 보안 패치 적용을 위해 커뮤니티 피드백이 가장 빠르게 반영되는 안정화 버전 선택

### 6. 사소한 트러블슈팅 기록

- **구성 파일 오류**: `docker-composer.yml` 파일명 오타 수정을 통해 컨테이너 배포 오류 해결
- **디렉토리 권한 거부**: 볼륨 접근 권한(EACCES) 이슈를 `chown` 명령어로 조정하여 데이터 영속성 확보
